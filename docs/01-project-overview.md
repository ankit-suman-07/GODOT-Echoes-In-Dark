# Project Overview

## Purpose

Develop a small but polished 2D horror game while learning fundamental game development concepts and producing a portfolio-quality project.

---

# Project Vision

Create a short horror experience where the player must explore, survive, and escape while avoiding a dangerous creature.

The emphasis is on:

* Atmosphere
* AI
* Exploration
* Resource management
* Clean architecture

---

# Objectives

## Gameplay

* Complete a playable horror game
* Maintain consistent tension
* Reward exploration

## Technical

* Learn Godot
* Implement AI using FSM
* Practice clean project architecture
* Improve Git workflow

---

# Scope

## Included

### Player

* Movement
* Sprint
* Interaction

### Gameplay

* Flashlight
* Batteries
* Keys
* Locked doors
* Story notes

### Enemy

* Patrol
* Detect player
* Chase
* Return to patrol

### UI

* Main menu
* Pause menu
* Win screen
* Lose screen

---

# Out of Scope

* Combat
* Inventory
* Multiple enemies
* Procedural generation
* Multiplayer
* Save system
* Dialogue
* Boss fights

---

# Gameplay Loop

```text
Explore
    ↓
Collect Resources
    ↓
Avoid Enemy
    ↓
Find Exit Key
    ↓
Unlock Exit
    ↓
Escape
```

---

# Success Criteria

The project is successful if:

* The game is fully playable.
* The gameplay loop is complete.
* Enemy AI behaves correctly.
* The project is documented.
* The game is polished enough for a portfolio.
